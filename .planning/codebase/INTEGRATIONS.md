# External Integrations

**Analysis Date:** 2026-05-14

## APIs and External Services

**MinerU Sidecar:**
- Used for strict document parsing and artifact capture.
- Client: `backend/src/ragstudio/services/mineru_client.py`.
- Endpoints used: `/health`, `/parse-async`, `/parse-jobs/{job_id}`, and `/parse-jobs/{job_id}/artifacts`.
- Configuration lives in runtime profile fields such as `mineru_enabled`, `mineru_base_url`, `mineru_timeout_ms`, and parse options.

**RAG-Anything and LightRAG Runtime:**
- Native runtime is built by `backend/src/ragstudio/services/runtime_factory.py`.
- Runtime imports `raganything` and `lightrag`, then creates `NativeRAGAnythingAdapter`.
- Product runtime policy currently requires `runtime_mode=runtime`, `storage_backend=postgres_pgvector_neo4j`, `parser_mode=mineru_strict`, and `embedding_provider=vllm_openai`.

**LLM, Embedding, Vision, and Reranker Providers:**
- Settings are stored in `SettingsProfile` and exposed through `/api/settings/default`.
- Connection checks live in `embedding_connection_service.py`, `llm_connection_service.py`, `reranker_connection_service.py`, and `mineru_client.py`.
- Reranker outbound hosts are constrained by `RAGSTUDIO_ALLOWED_RERANKER_HOSTS` and `AppSettings.allowed_reranker_hosts`.
- Provider manifest preview is implemented in `backend/src/ragstudio/services/provider_manifest_service.py`.

## Data Storage

**PostgreSQL with PGVector:**
- Primary metadata and vector storage.
- Compose image: `pgvector/pgvector:pg17` in `docker-compose.yml`.
- Async SQLAlchemy engine/session setup in `backend/src/ragstudio/db/engine.py`.
- Models include documents, chunks, index records, graph projection records, jobs, variants, evaluation sets, experiments, runs, scores, and optimization sessions.

**Neo4j:**
- Graph projection and graph inspection backend.
- Compose image: `neo4j:5-community`.
- Graph materialization lives in `backend/src/ragstudio/services/graph_materialization_service.py` and runner logic in `graph_projection_runner.py`.
- Graph workspace identifiers are normalized in `backend/src/ragstudio/services/graph_workspace.py`.

**Local Artifact Store:**
- Uploaded files and parser artifacts are stored under `RAGSTUDIO_DATA_DIR`.
- Artifact path handling is centralized in `backend/src/ragstudio/services/artifact_store.py`.
- Document delete cleanup also removes related index and graph artifacts through `DocumentService`.

## Authentication and Identity

- No user authentication provider is present in this codebase.
- The current app is a local/operator-style Studio workbench, not a multi-tenant hosted product.
- Public launch work should avoid exposing the live backend or upload surface unless a separate auth/security design is added.

## Monitoring and Observability

**Runtime Diagnostics:**
- `/api/diagnostics` is served by `backend/src/ragstudio/api/routes/diagnostics.py`.
- Health checks are generated by `RuntimeHealthService`.
- Query traces, chunk traces, reranker traces, token metadata, timings, and errors are persisted on `Run`.

**Worker Observability:**
- The indexing worker runs `python -m ragstudio.workers.index_worker`.
- Worker healthcheck calls `python -m ragstudio.workers.index_worker --healthcheck`.
- Job status, progress, logs, lease fields, heartbeat, and recovery action are stored in `Job`.

**Logging:**
- Logging is configured at app creation through `backend/src/ragstudio/logging.py`.
- Docker services use stdout/stderr logs.

## CI/CD and Deployment

**Local Development:**
- `./scripts/setup.sh` builds backend and frontend images.
- `./scripts/dev.sh` starts Docker Compose.
- `./scripts/test-all.sh` builds images, starts storage services, then runs backend and frontend validation.

**CI Status:**
- No `.github/workflows` files were found during mapping.
- Test and build gates are script-driven rather than repository CI-driven.

**Public Launch Site Boundary:**
- The approved proof-system plan expects a separate `ragstudio-site` repository for Cloudflare Pages.
- This Ragstudio repo should own benchmark generation/export and proof packet source artifacts; the site imports and renders those artifacts.

## Environment Configuration

**Development:**
- Example environment names are in `.env.example`.
- Docker Compose overrides backend/worker settings through service-level environment variables.
- Frontend dev server proxies `/api` and `/openapi.json` to the backend target.

**Production-Like Local Stack:**
- Storage and runtime settings are configured through environment variables and persisted settings profiles.
- Public proof artifacts must redact private endpoints, private hostnames, API keys, and local paths before export.

## Webhooks and Callbacks

- No incoming webhook endpoints were found.
- Outgoing provider calls are direct HTTP requests from backend services.
- MinerU parsing is asynchronous polling, not webhook callback based.
