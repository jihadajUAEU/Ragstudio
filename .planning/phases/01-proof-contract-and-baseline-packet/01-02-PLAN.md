<threat_model>
Assets: packet schema contracts, public claim statuses, claim-to-artifact evidence links, and compatibility metadata.
Threats: loose schemas accepting unsafe private evidence, claims marked `proven` without artifact evidence, hidden disabled/roadmap claims, and field-name drift before Phase 2 validation.
Mitigations: JSON Schema 2020-12 contracts with `required`, `enum`, and `additionalProperties: false`; claim status enum; evidence-path requirements for `proven`; explicit missing-evidence fields for `roadmap` and `disabled`.
Residual risk: executable schema validation is deferred to Phase 2; Phase 1 should keep schemas strict and JSON parseable.
</threat_model>
