<threat_model>
Assets: runtime trust state, provider connectivity results, saved settings
profile, secret presence flags, and every Studio route header.
Threats: misleading `Ready` state when diagnostics are unavailable, polling
provider test endpoints by accident, leaking API key values from settings,
overlapping header controls on mobile, focus trapped behind the panel, or user
mistaking an unsaved Settings form for the tested runtime profile.
Mitigations: deterministic frontend status mapping, diagnostics-only polling,
saved default profile only, no rendering of secret fields, independent provider
mutations triggered only by explicit buttons, compact chip styling, and
focus-trapped dialog tests.
Residual risk: exact provider failure shapes depend on diagnostics payloads; if
existing payload cannot deterministically identify provider issues, add the
smallest backend diagnostics field in a follow-up task before verification.
</threat_model>
