<threat_model>
Assets: public proof packet artifacts, schemas, claims registry, screenshot
signoff metadata, redaction policy, and structured validation output.
Threats: schema bypass, path traversal outside the packet root, hash spoofing,
private content leaks, proven claims without public evidence, hidden roadmap or
disabled claims, stale packet metadata, and tests that duplicate validation
logic instead of exercising it.
Mitigations: JSON Schema 2020-12 validation through `jsonschema`, root-relative
path resolution that rejects absolute and `..` paths, SHA-256 recomputation,
fail-closed text-file redaction scanning, claim-count/evidence validation,
screenshot signoff validation, stable finding codes, and temp-packet mutation
tests.
Residual risk: CLI formatting, export-manifest generation, and full recovery
docs are handled in Plans 02 and 03.
</threat_model>
