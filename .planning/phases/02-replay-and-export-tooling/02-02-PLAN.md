<threat_model>
Assets: first-time evaluator trust path, CLI exit codes, machine-readable JSON
validation output, export manifest metadata, and recovery documentation.
Threats: command accidentally requiring Docker/providers/backend, JSON output
including local/private paths, strict mode ignoring warnings, human output hiding
real failures, export metadata recording stale hashes or source state, and docs
overstating replay guarantees.
Mitigations: thin Bash wrapper with `PYTHONPATH=backend/src`, no runtime service
imports, compact JSON using validator result only, explicit warning-to-failure
strict mode, subprocess CLI tests, static packet export helper, and docs that
separate static replay from future/live capture.
Residual risk: broader mutation/failure-path matrix is expanded in Plan 03.
</threat_model>
