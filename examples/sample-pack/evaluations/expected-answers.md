# Expected Answer Behavior

These examples are synthetic and not copied from any source.

## Answer With Evidence

- Protocol questions should cite section labels such as `SP-1.2` and avoid inventing protocol fields.
- Financial questions should preserve table numbers and fiscal periods from `FIN-2026-R2`.
- Regulatory questions should ground answers in agency, action, date, and docket-like labels.
- Scientific questions should distinguish methods, figure captions, citations, and unsupported conclusions.
- Technical report questions should preserve units such as `L/min` and acronyms such as `ATL`.
- Book questions should preserve chapter and footnote labels, including multilingual phrases.
- Legal opinion questions should cite synthetic case sections and invented citation relationships.
- OCR stress questions should expose parser warnings before treating broken rows as evidence.

## Evidence Format

Good answers should include the document ID, reference label, and short cited span. If Ragstudio cannot retrieve the exact reference, the answer should say that evidence is missing.
